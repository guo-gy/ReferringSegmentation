# 广义指代图像分割 (G-RIS) 毕业设计执行计划 (冲刺版)

本计划专为在 6-8 周内高效完成“基于跨模态对齐的广义指代图像分割算法研究及应用”毕业设计而制定。

## 一、 系统架构选型 (最高效方案)

*   **数据集**: `gRefCOCO` (核心，满足“广义”要求) + `RefCOCO/RefCOCOg` (对比验证基准)。
*   **骨干网络 (Backbone)**: `CLIP (ViT-B/16 + Text Encoder)`。利用预训练的 CLIP 实现降维打击，解决跨模态特征提取和初步对齐。
*   **基线网络 (Baseline)**: 参考 `CRIS (CLIP-Driven Referring Image Segmentation)` 或最新针对 `gRefCOCO` 的开源方案。
*   **创新方向**: 聚焦在“文本-视觉融合层 (Fusion Layer)”。建议加入多模态自蒸馏机制或注意力特征池化操作。
*   **演示系统**: `Gradio` 或 `Streamlit` (几行纯 Python 代码即可实现精美 Web 界面)。

## 二、 硬件资源预估与要求

*   **推荐显卡**: RTX 3090 (24GB VRAM) 或更高，体验最佳。
*   **可接受显卡**: RTX 4080 (16GB VRAM) 等，适当调整 Batch Size。
*   **丐版方案**: 8GB-12GB VRAM (如 RTX 3060/4060)，必须开启 AMP (混合精度)、Gradient Accumulation (梯度累加)，并将 Batch Size 降至极低 (2-4)。
*   **单次训练耗时估计**: 单卡 RTX 3090 下，单个数据集跑完全程约需 **15 - 30 小时** (取决于数据集复杂度和具体改进模块)。
