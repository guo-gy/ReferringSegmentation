# 广义指代图像分割 (G-RIS) 毕业设计执行计划 (冲刺版)

本计划专为在 6-8 周内高效完成“基于跨模态对齐的广义指代图像分割算法研究及应用”毕业设计而制定。

## 一、 系统架构选型 (最高效方案)

*   **数据集**: `gRefCOCO` (核心，满足“广义”要求) + `RefCOCO/RefCOCOg` (对比验证基准)。
*   **骨干网络 (Backbone)**: 
    *   **推荐方案 (稳妥极速)**: `CLIP (ViT-B/16 或 ViT-L/14)` + `CLIP Text Encoder`。甚至 2025 年的 SOTA 模型 (如 OMG-Seg) 仍在大量使用冻结的 CLIP 作为跨模态提取器，降维打击效果极好。
    *   **备选经典方案**: `Swin-Transformer (Swin-B/L)` + `RoBERTa`。gRefCOCO 原论文的基础代码 (如 ReLA) 大量使用此架构，视觉特征提取能力极强。
    *   **2025 绝对前沿 (供论述展望，不建议实操)**: 多模态大语言模型 (MLLM, 如 LLaVA/Qwen-VL)。最新的 UniRES++ (2025) 等模型已切换至此架构，但算力要求极高。
*   **基线网络 (Baseline)**: 参考 `CRIS` 或 gRefCOCO 的官方 Baseline `ReLA`开源方案。
*   **创新方向**: 聚焦在“文本-视觉融合层 (Fusion Layer)”。建议加入多模态自蒸馏机制、注意力特征池化，或对象级推理 (Object-level Reasoning，参考 2024 年底的 InstAlign 思想)。
*   **演示系统**: `Gradio` 或 `Streamlit` (几行纯 Python 代码即可实现精美 Web 界面)。

## 二、 硬件资源预估与要求 (已确认具备 A100/H100 顶级算力)

*   **当前可用显卡**: **A100 (80GB) / H100 (80GB+)**。这是目前的工业级/科研级顶级算力天花板。
*   **算力红利**: 极大的显存允许你将 `Batch Size` 开到最大 (如 128 或 256)，不仅能成倍加速对比学习的收敛，还能极大提升模型的各项精度指标！
*   **单次训练耗时估计**: 在单张 A100/H100 加持下，原本需要 30 小时的全量数据集训练，预计可压缩至 **4 - 8 小时** 跑完。
*   **极限并行打法**: 如果你申请到的是多卡节点 (如 4x A100)，你可以利用 `torch.distributed` 同时开 4 个容器，**并行跑 Baseline 和 3 个由于你的创新点产生的消融模型**。原本需要 10 天的实验周期，在这等算力下 **一个下午 (4-6小时)** 即可全部跑完！
